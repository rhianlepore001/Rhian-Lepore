## Language & Types

- **TypeScript strict mode** — `tsconfig.json` enforces strict typing
- **Named exports** — All pages and components use named exports: `export const Dashboard`
- **Interface over type** — Use `interface` for component props, `type` for unions/primitives
- **No `any`** — ESLint enforced, zero `any` tolerance in strict config


## Supabase Patterns

### Client Import
```tsx
import { supabase } from '@/lib/supabase';
```

### Auth Context Usage
```tsx
const { user, profile, company_id, role } = useAuth();
```

### Data Query Pattern
```tsx
const { data, error } = await supabase
  .from('table_name')
  .select('*')
  .eq('company_id', company_id);

if (error) {
  console.error('Error:', error);
  return;
}
```

### RPC Call Pattern
```tsx
const { data, error } = await supabase
  .rpc('function_name', { 
    p_company_id: company_id,
    p_start_date: startDate,
  });
```

## Multi-Tenancy Rules

> ⚠️ **Critical** — Never hardcode `company_id`. Always source from `useAuth()` hook.

```tsx
// ✅ Correct
const { company_id } = useAuth();
await supabase.from('appointments').select('*').eq('company_id', company_id);

// ❌ Wrong — never do this
await supabase.from('appointments').select('*').eq('company_id', 'some-uuid');
```

## Error Handling

### Async Operations
```tsx
try {
  const { data, error } = await supabase.from('...').select();
  if (error) throw error;
  setData(data);
} catch (err) {
  console.error(err);
  addAlert({ type: 'error', message: 'Erro ao carregar dados' });
} finally {
  setLoading(false);
}
```

### Error Boundary
- `components/ErrorBoundary.tsx` wraps the app for unhandled render errors

## Naming Conventions

| Type | Convention | Example |
|------|-----------|---------|
| Components | PascalCase | `AppointmentWizard` |
| Hooks | camelCase `use` prefix | `useDashboardData` |
| Contexts | PascalCase + Context suffix | `AuthContext` |
| Files (components) | PascalCase | `TeamMemberForm.tsx` |
| Files (hooks) | camelCase | `useDashboardData.ts` |
| Constants | SCREAMING_SNAKE_CASE | `MAX_RETRY_COUNT` |
| DB table refs | snake_case | `company_id`, `created_at` |
| Route paths | kebab-case Portuguese | `/configuracoes/equipe` |

## Styling Conventions

- **Tailwind utilities** — Primary styling method
- **No inline styles** — Use `className` with Tailwind
- **Dark theme base** — `bg-neutral-900`, `text-white`
- **Dynamic branding** — `--color-primary`, `--color-accent` CSS vars via `DynamicBranding`
- **Responsive** — Mobile-first, `sm:`, `md:`, `lg:` breakpoints
- **Design system** — Custom "Brutal" aesthetic: `BrutalCard`, `BrutalButton`, `BrutalBackground`
