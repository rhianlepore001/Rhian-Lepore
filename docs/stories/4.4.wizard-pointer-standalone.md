# US-0404: WizardPointer standalone via GuidedModeContext

**Epic:** EPIC-004 — Onboarding Wizard Refactor
**Prioridade:** P1 | **Blocker:** US-0402
**Esforço:** 3h
**Lead:** @dev
**Objetivo:** Extrair o `WizardPointer` do acoplamento com `WizardOverlay`/`WizardEngine` e torná-lo um componente standalone que renderiza via `ReactDOM.createPortal` e se conecta ao `GuidedModeContext`.

---

## 📖 História

**Como um** usuário no dashboard seguindo o tour guiado,
**Eu quero** ver a seta indicadora em qualquer página do sistema — não apenas na tela de onboarding —
**Para que** o wizard me guie independente de onde eu estiver no AgenX.

---

## Contexto Técnico

### Estado atual
```
WizardEngine.tsx
  └── WizardOverlay.tsx  (overlay full-screen, z-index 9997)
        └── WizardPointer.tsx  (z-index 9999, recebe `target` como prop)
```
O `WizardPointer` depende do `WizardEngine` para receber `target`. Fora do overlay, ele não funciona.

### Estado proposto
```
App.tsx
  └── GuidedModeProvider
        └── StandaloneWizardPointer  (novo wrapper, sempre montado, só renderiza quando ativo)
              └── WizardPointer  (lê estado do GuidedModeContext via portal no body)
```

### Implementação do wrapper

```typescript
// components/onboarding/StandaloneWizardPointer.tsx
import { useGuidedMode } from '@/contexts/GuidedModeContext';
import { WizardPointer } from './WizardPointer';
import ReactDOM from 'react-dom';

export function StandaloneWizardPointer() {
  const { isGuideActive, targetElementId, position, message } = useGuidedMode();

  if (!isGuideActive || !targetElementId) return null;

  return ReactDOM.createPortal(
    <WizardPointer
      target={{ elementId: targetElementId, position, message }}
    />,
    document.body
  );
}
```

### Preservar comportamento existente do WizardPointer
O componente `WizardPointer.tsx` já tem toda a lógica necessária:
- Spotlight via `boxShadow: 0 0 0 9999px rgba(0,0,0,0.75), 0 0 0 4px #F59E0B`
- `MutationObserver` para aguardar elemento aparecer no DOM
- Modo mobile: pulse circle
- `scrollIntoView` automático

**Não alterar** a lógica interna de `WizardPointer.tsx` — apenas adicionar suporte ao uso via Context.
