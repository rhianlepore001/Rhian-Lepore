# US-0407: Retomada de sessão do wizard

**Epic:** EPIC-004 — Onboarding Wizard Refactor
**Prioridade:** P2 | **Blockers:** US-0405, US-0406
**Esforço:** 3h
**Lead:** @dev
**Objetivo:** Quando o usuário fecha o browser no meio do tour e retorna, oferecer a retomada do wizard exatamente no step onde parou.

---

## 📖 História

**Como um** usuário que interrompeu o tour de configuração no meio do caminho,
**Eu quero** que o sistema lembre onde parei e me ofereça continuar de onde estava,
**Para que** eu não precise recomeçar o tour do início e encontre rapidamente o que ainda precisa ser configurado.

---

## Contexto Técnico

### Fonte de dados
O campo `onboarding_progress.step_data.last_guided_step` (gravado em US-0406) contém o ID do último step onde o guided mode estava ativo.

### Lógica de detecção de retomada
```typescript
const shouldShowResumeBanner =
  !!stepData?.last_guided_step &&
  !!stepData?.guided_started &&
  !stepData?.guided_dismissed_at &&
  !checks[checkKeyFor(stepData.last_guided_step)]; // step ainda não concluído
```

### Copy do banner de retomada
```
"Você parou em [Nome do Step] — quer continuar?"
Botões: [Continuar →]  [Não, obrigado]
```

Exemplos de nome do step:
- `services` → "Cadastrar Serviços"
- `hours` → "Configurar Horários"
- `profile` → "Personalizar Perfil"

### Distinção visual do banner de primeira visita
| Aspecto | Banner 1ª visita | Banner de retomada |
|---------|------------------|--------------------|
| Ícone | 🚀 Rocket | 🔄 RefreshCw |
| Copy | "Quer um tour guiado?" | "Você parou em [Step]..." |
| Cor de fundo | accent-gold/10 | blue-500/10 |

### Botões
- **"Continuar":** `startGuide(last_guided_step)` + navega
- **"Não, obrigado":** remove `last_guided_step` do `step_data` → atualiza Supabase
