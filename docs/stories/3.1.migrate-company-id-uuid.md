# US-0301: Migrar `company_id` TEXT → UUID em profiles

**Epic ID:** EPIC-003-S1 (Sprint 1 — Database Security & Accessibility Foundation)
**Prioridade:** P0 (Crítico)
**Esforço:** 8 horas
**Lead:** Dara (@data-engineer)
**Objetivo:** Eliminar risco de isolamento de dados — corrigir incompatibilidade de tipo TEXT vs UUID que bloqueia políticas RLS

---

## 📖 História

**Como um** data engineer focado em segurança multi-tenant,
**Eu quero** migrar a coluna `company_id` de TEXT para UUID em todas as tabelas relacionadas,
**Para que** possamos aplicar políticas RLS consistentes e eliminar vulner abilidades de vazamento de dados entre tenants.

## 🚨 BLOCKER — Análise de Impacto (2026-03-20)

**Status:** BLOCKED — Risco crítico identificado via MCP antes da execução.

**Achados via auditoria MCP:**
- `get_auth_company_id()` é chamada em **9 policies de 7 tabelas** (appointments, business_settings, clients, finance_records, profiles, public_bookings, services)
- Todas as comparações usam `user_id (TEXT) = get_auth_company_id()` — se o retorno mudar para UUID, quebra 9 policies simultaneamente
- Postgres **NÃO tem cast implícito** entre TEXT e UUID → lockout total de todos os usuários

**Plano de execução segura (requer janela de manutenção):**
1. Dropar as 9 policies dependentes
2. ALTER TABLE profiles ALTER COLUMN company_id TYPE uuid USING company_id::uuid
3. DROP FUNCTION get_auth_company_id() / recriar com retorno UUID
4. Atualizar todos os user_id em outras tabelas OU usar comparação `user_id::uuid = get_auth_company_id()`
5. Recriar as 9 policies com cast correto
6. Validar em staging antes de prod

**Estimativa revisada:** 8h → 12h (incluindo rollback plan e staging)

## 📝 Dev Notes

### Fonte Arquitetural
**Documento:** `docs/architecture/db-specialist-review.md` (Section 1: Schema Normalization, Section 3: RLS & Security)

**Informações Críticas:**
- **Problema:** Coluna `company_id` em `profiles` é TEXT, mas outras tabelas usam UUID
- **Impacto:** RLS policies não conseguem comparar TEXT com UUID, permitindo vazamento de dados
- **Solução:** Converter TEXT → UUID com integridade de dados garantida
- **Precedência:** Este é o blocker P0 — todos os outros trabalhos de RLS dependem disso

### SQL Statements Relevantes

```sql
-- Verificar tipo atual
SELECT column_name, data_type
FROM information_schema.columns
WHERE table_name = 'profiles' AND column_name = 'company_id';

-- Migração segura
BEGIN TRANSACTION;
  ALTER TABLE profiles
  ALTER COLUMN company_id TYPE uuid USING company_id::uuid;
COMMIT;

-- Validação pós-migração
SELECT COUNT(*) as total_rows,
       COUNT(DISTINCT company_id) as unique_companies,
       COUNT(CASE WHEN company_id IS NULL THEN 1 END) as null_companies
FROM profiles;

-- Índice crítico
CREATE INDEX idx_profiles_company_id ON profiles(company_id);
ANALYZE profiles;
```
