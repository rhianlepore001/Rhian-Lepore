## Contexto Técnico

### Detecção do Activation Event
O evento `setup-step-completed` com `stepId: 'appointment'` (disparado em US-0408 após criar agendamento) é o gatilho. Mas deve verificar se é realmente o **primeiro** agendamento:

```typescript
const isFirstAppointment = appointmentsCount === 1; // verificar antes de salvar
```

### Banner de celebração

```
"Seu primeiro agendamento foi criado! 🚀"
Subtítulo: "O AgenX está no ar. Agora você pode acompanhar tudo pelo dashboard."
Botão: [Ver meu Dashboard →]
Auto-dismiss: 8 segundos (ou ao clicar no X)
```

Posição: banner flutuante no topo da tela, acima do conteúdo (similar ao `SmartNotificationsBanner`).

### Persistência do estado ativado

```typescript
// Ao disparar o Activation Event:
await supabase.from('profiles').update({
  activation_completed: true,
  activated_at: new Date().toISOString(),
}).eq('id', user.id);

// onboarding marcado como completo:
await completeOnboarding(companyId); // já existe em lib/onboarding.ts
```

### SetupCopilot em estado "ativado"

Quando `activation_completed = true`:
- Substituir lista de steps por mensagem final:
```

Link: Ver relatórios / Ver agenda
```
- SetupCopilot some completamente após 7 dias da ativação

## ⚠️ Pré-condição de Banco (verificar antes de implementar)

> **Ação obrigatória antes do desenvolvimento:** Verificar se as colunas `activation_completed` e `activated_at` existem na tabela `profiles` no Supabase.
>
> Comando de verificação:
> ```sql
> SELECT column_name FROM information_schema.columns
> WHERE table_name = 'profiles'
> AND column_name IN ('activation_completed', 'activated_at');
> ```
>
> Se não existirem → criar migration antes de iniciar a story:
> ```sql
> ALTER TABLE profiles
>   ADD COLUMN IF NOT EXISTS activation_completed BOOLEAN DEFAULT false,
>   ADD COLUMN IF NOT EXISTS activated_at TIMESTAMPTZ;
> ```
