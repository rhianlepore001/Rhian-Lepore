# US-0410: Animações e polish do wizard

**Epic:** EPIC-004 — Onboarding Wizard Refactor
**Prioridade:** P2 | **Blockers:** US-0406, US-0408
**Esforço:** 4h
**Lead:** @dev
**Objetivo:** Garantir que todas as animações do wizard sejam fluidas, acessíveis (`prefers-reduced-motion`) e performáticas em dispositivos móveis com hardware limitado.

---

## 📖 História

**Como um** usuário do AgenX em qualquer dispositivo,
**Eu quero** que as animações do wizard sejam suaves e não causem travamentos,
**Para que** a experiência guiada seja agradável tanto no celular quanto no desktop, sem distrair do objetivo principal.

---

## Especificações de Animação

### Tabela de tokens de motion

| Elemento | Animação | Duração | Easing | Trigger |
|----------|----------|---------|--------|---------|
| WizardPointer entrada | fade-in + scale 0.8 → 1.0 | 200ms | ease-out | `isGuideActive` → true |
| WizardPointer saída | fade-out + scale 1.0 → 0.8 | 150ms | ease-in | `endGuide()` |
| WizardPointer bounce (desktop) | `wizard-pointer-bounce` | 1s | ease-in-out infinite | enquanto ativo |
| WizardPointer pulse (mobile) | `wizard-pulse` | 1.5s | ease-in-out infinite | enquanto ativo |
| Check concluído (SetupCopilot) | scale 0 → 1.2 → 1.0 | 300ms | spring (cubic-bezier) | step concluído |
| Banner entrada | slide-in-from-top + fade | 250ms | ease-out | banner mount |
| Banner saída | slide-out-to-top + fade | 200ms | ease-in | banner dismiss |
| Celebration banner | slide-in-from-top + scale 0.95→1 | 350ms | spring | activation event |

### CSS para check concluído (SetupCopilot)

```css
@keyframes check-complete {
  0%   { transform: scale(0); opacity: 0; }
  60%  { transform: scale(1.2); opacity: 1; }
  100% { transform: scale(1); }
}

.animate-check-complete {
  animation: check-complete 300ms cubic-bezier(0.34, 1.56, 0.64, 1) forwards;
}
```

## Regras de Acessibilidade

### `prefers-reduced-motion`

```css
@media (prefers-reduced-motion: reduce) {
  /* Substituir bounce/pulse por fade estático */
  [class*="wizard-pointer-bounce"],
  [class*="wizard-pulse"] {
    animation: none !important;
    opacity: 0.9;
  }

  /* Encurtar todas as transições para 0 ou mínimo */
  .animate-check-complete {
    animation-duration: 50ms !important;
  }
}
```

### Performance mobile

```typescript
// Detectar hardware limitado e desativar animações infinitas
const isLowEndDevice = navigator.hardwareConcurrency <= 4;

// Aplicar classe no root para override de animações
if (isLowEndDevice) {
  document.documentElement.classList.add('low-end-device');
}
```

```css
.low-end-device [class*="animate-[wizard-pointer-bounce"] {
  animation: none;
  opacity: 0.85;
}
```

### Regra de `will-change`

```
Usar will-change APENAS no WizardPointer (1 elemento por vez).
NUNCA adicionar will-change em banners, SetupCopilot, ou outros elementos.
Máximo 3 elementos com will-change simultâneos na página.
```
