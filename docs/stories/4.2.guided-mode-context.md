**Objetivo:** Criar o Context React que gerencia o estado do wizard guiado, permitindo que o WizardPointer persista entre navegações de rota.

## 📖 História

**Como um** usuário que iniciou o tour guiado do SetupCopilot,
**Eu quero** que a seta indicadora continue aparecendo mesmo quando navego para outra página,
**Para que** eu não perca o contexto do que preciso fazer ao ser redirecionado para as páginas de configuração.

---

## Contexto Técnico

### Por que um Context global é necessário
O `WizardPointer` atual está acoplado ao `WizardEngine` e só funciona dentro do `WizardOverlay`. Quando o usuário navega (ex: do dashboard para `/configuracoes/agendamento`), o componente é desmontado e o estado do pointer se perde.

O `GuidedModeContext` resolve isso mantendo o estado do wizard ativo na raiz da aplicação, independente da rota atual.

### Interface do Context

```typescript
// contexts/GuidedModeContext.tsx

interface GuidedModeState {
  activeStep: WizardStepId | null;        // ex: "hours", "services"
  targetElementId: string | null;          // ex: "business-hours-section"
  position: 'top' | 'bottom' | 'left' | 'right';
  message: string;
  isGuideActive: boolean;
}

interface GuidedModeContextValue extends GuidedModeState {
  startGuide: (stepId: WizardStepId) => void;
  endGuide: () => void;
}
```

### Estratégia de persistência
- **sessionStorage:** persiste durante a sessão (sobrevive a navegações, perde ao fechar o browser)
- **onboarding_progress.step_data:** persiste entre sessões (campo `last_guided_step`)
- Ao inicializar o app: verificar `sessionStorage` e restaurar estado pendente se existir
