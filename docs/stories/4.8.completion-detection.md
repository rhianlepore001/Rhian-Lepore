# US-0408: Completion detection por página

**Epic:** EPIC-004 — Onboarding Wizard Refactor
**Prioridade:** P1 | **Blocker:** US-0406
**Esforço:** 4h
**Lead:** @dev
**Objetivo:** Fazer com que o WizardPointer detecte automaticamente quando o usuário completou a ação na página e se auto-dispense com animação de sucesso, sem precisar que o usuário volte manualmente ao dashboard.

---

## 📖 História

**Como um** usuário que acabou de salvar suas configurações de horário,
**Eu quero** que a seta indicadora desapareça automaticamente e me mostre que completei o passo,
**Para que** eu saiba que posso voltar ao dashboard e ver o progresso atualizado.

---

## Contexto Técnico

### Mecanismo: Custom Event

Cada página-alvo, após um save bem-sucedido, dispara:
```typescript
window.dispatchEvent(
  new CustomEvent('setup-step-completed', { detail: { stepId: 'hours' } })
);
```

### Consumidores do evento

**1. StandaloneWizardPointer (US-0404)**
```typescript
useEffect(() => {
  const handler = (e: CustomEvent) => {
    if (e.detail.stepId === activeStep) {
      // Exibir ícone de check por 1500ms, depois endGuide()
      showSuccessAnimation().then(() => endGuide());
    }
  };
  window.addEventListener('setup-step-completed', handler);
  return () => window.removeEventListener('setup-step-completed', handler);
}, [activeStep]);
```

**2. SetupCopilot**
```typescript
useEffect(() => {
  const handler = () => checkSetupProgress(); // re-query Supabase
  window.addEventListener('setup-step-completed', handler);
  return () => window.removeEventListener('setup-step-completed', handler);
}, []);
```

### Páginas a modificar — onde disparar o evento

| Página | Quando disparar |
|--------|----------------|
| ServicesSettings | Após salvar/criar serviço com sucesso |
| TeamSettings | Após salvar/criar membro com sucesso |
| BusinessHoursSettings | Após salvar horários com sucesso |
| ProfileSettings | Após salvar logo/capa com sucesso |
| PublicBookingSettings | Após ativar toggle + salvar |
| Agenda | Após criar primeiro agendamento (US-0409 usa isso também) |

### Regra crítica
O evento **NÃO deve ser disparado em caso de erro.** Apenas em `try { ... sucesso ... dispatchEvent() }`, nunca no `catch`.

## ✅ Critérios de Aceitação

- [ ] **AC1:** Evento `setup-step-completed` disparado após save bem-sucedido em cada uma das 6 páginas
- [ ] **AC2:** Evento NÃO disparado quando o save falha com erro
- [ ] **AC3:** WizardPointer exibe ícone de check (✓) por 1500ms ao receber o evento do step ativo
- [ ] **AC4:** Após 1500ms, WizardPointer executa animação de saída e `endGuide()` é chamado
- [ ] **AC5:** SetupCopilot re-executa `checkSetupProgress()` ao receber qualquer evento de conclusão
- [ ] **AC6:** Check verde animado aparece no step correspondente no SetupCopilot
- [ ] **AC7:** Evento disparado apenas quando `detail.stepId` é um `WizardStepId` válido
- [ ] **AC8:** Após `endGuide()` automático, usuário **permanece na página atual** — não é redirecionado
- [ ] **AC9:** Após WizardPointer desaparecer, exibir toast/snackbar: `"✓ [Nome do Step] concluído! Próximo: [Nome do Próximo Step Pendente]"` com link para voltar ao dashboard
- [ ] **AC10:** Se não houver próximo step pendente, toast exibe: `"Você está quase lá! Voltar ao dashboard →"`
