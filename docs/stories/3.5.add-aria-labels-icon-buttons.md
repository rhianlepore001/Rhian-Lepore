# US-0305: Adicionar labels ARIA em botões de ícone

**Epic ID:** EPIC-003-S1 (Sprint 1)
**Prioridade:** P1 (Quick Win)
**Esforço:** 3 horas
**Lead:** Uma (@ux-design-expert)
**Objetivo:** Usuários de leitores de tela conseguem identificar botões de ícone em 15+ componentes

---

## 📖 História

**Como um** usuário de leitor de tela (NVDA, JAWS),
**Eu quero** ouvir descrições significativas de botões de ícone,
**Para que** eu possa navegar e interagir com a interface sem confusão.

---

## ✅ Critérios de Aceitação

- [x] **AC1:** Mínimo 15 botões de ícone têm `aria-label` descritivo
- [x] **AC2:** Labels são significativos (ex: "Settings" não "gear icon")
- [ ] **AC3:** Testad com leitor de tela em Chrome/Firefox (NVDA no Windows)
- [x] **AC4:** Nenhuma quebra visual ou funcional

## 📝 Dev Notes

**Implementação Padrão:**
```tsx
// Before
<button onClick={handleSettings}>
  <Gear size={20} />
</button>

// After
<button
  aria-label="Open settings"
  onClick={handleSettings}
>
  <Gear size={20} />
</button>
```

**Componentes a Atualizar:**
- `Header.tsx` — Notifications, Profile, Logout buttons
- `Sidebar.tsx` — Menu toggle, collapse buttons
- `Modal.tsx` — Close button (X icon)
- `Card.tsx` — Edit, delete, more buttons
- Navigation buttons throughout app

**Testing with NVDA:**
1. Baixar NVDA (gratuito): https://www.nvaccess.org/
2. Ativar: Ctrl + Alt + N
3. Navegar com Tab
4. Ouvir aria-label pronunciadas
