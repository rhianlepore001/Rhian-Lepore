# US-0401: Simplificar Onboarding Inicial

**Epic:** EPIC-004 — Onboarding Wizard Refactor
**Prioridade:** P0 (sem blocker — pode iniciar imediatamente, paralela com US-0402)
**Esforço:** 5h
**Lead:** @dev
**Objetivo:** Remover o WizardEngine overlay bloqueante e os steps que duplicam dados já coletados no registro, substituindo por uma tela de boas-vindas simples + 1 step essencial.

---

## 📖 História

**Como um** novo usuário que acabou de se registrar no AgenX,
**Eu quero** ser recebido por uma tela simples e direta que me leva ao dashboard rapidamente,
**Para que** eu possa começar a usar o sistema sem ser bloqueado por formulários que pedem informações que já forneci no cadastro.

---

## Contexto Técnico

### Problema atual
O `WizardEngine` (`components/onboarding/WizardEngine.tsx`) renderiza 5 steps via `WizardOverlay` (overlay full-screen bloqueante):
- **Step 1 (StepBusinessInfo):** Pede `fullName`, `businessName`, `phone` — **dados já coletados em `Register.tsx`** e salvos em `profiles`
- **Step 3 (StepTeam):** Cadastro de equipe — não essencial para o sistema funcionar
- **Step 4 (StepMonthlyGoal):** Meta mensal — sem contexto real (usuário acabou de registrar)

### Único step essencial
**Step 2 (StepServices):** Sem ao menos 1 serviço cadastrado, o sistema de agendamento não funciona. Este é o único step que deve permanecer obrigatório.

### Fluxo proposto
```
Register → Tela Boas-vindas (nome pré-preenchido, botão "Configurar primeiro serviço")
         → StepServices (1 serviço mínimo)
         → Redirect para /#/ (dashboard com SetupCopilot ativo)
```

---

## ✅ Critérios de Aceitação

- [x] **AC1:** Onboarding exibe apenas 1 tela de boas-vindas + StepServices (2 etapas no máximo)
- [x] **AC2:** Tela de boas-vindas exibe `businessName` pré-preenchido do perfil (sem campo editável)
- [x] **AC3:** `StepBusinessInfo` não é mais exibido no fluxo de onboarding
- [x] **AC4:** `StepTeam` e `StepMonthlyGoal` não são mais exibidos no fluxo obrigatório
- [x] **AC5:** Após cadastrar 1 serviço, usuário é redirecionado para `/#/` automaticamente
- [x] **AC6:** `WizardOverlay` não bloqueia interação com o dashboard após o redirect
- [ ] **AC7:** Rota `/onboarding` protegida — usuário sem onboarding concluído é redirecionado de volta
- [x] **AC8:** `onboarding_progress` atualizado corretamente ao concluir o novo fluxo simplificado
