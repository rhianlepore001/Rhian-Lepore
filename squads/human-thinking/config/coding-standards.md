# Coding Standards — Human Thinking Squad

## Padrões de Saída

### Relatórios
- Formato: Markdown estruturado
- Linguagem: Português (BR)
- Salvos em: `docs/thinking-cycles/`
- Naming: `{data}-{area}-{tipo}.md`

### Código Reformulado
- Seguir padrões existentes do projeto
- TypeScript strict (sem `any`)
- Tailwind CSS para estilos
- Functional components com hooks

### Priorização
- Usar sempre matriz impacto x esforço
- Quick wins primeiro
- Documentar trade-offs de cada decisão

## Regras dos Agentes
1. Sempre ler o código antes de opinar
2. Basear análises em dados, não suposições
3. Respeitar decisões documentadas (ADRs)
4. Não propor mudanças de stack sem justificativa forte
5. Manter foco no usuário final (owner, profissional, cliente)
