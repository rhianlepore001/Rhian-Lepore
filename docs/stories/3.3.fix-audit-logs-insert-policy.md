# US-0303: Corrigir política INSERT em `audit_logs` (verificação `user_id`)

**Epic ID:** EPIC-003-S1 (Sprint 1)
**Prioridade:** P0 (Crítico)
**Esforço:** 2 horas
**Lead:** Dara (@data-engineer)
**Objetivo:** Impedir falsificação de trilha de auditoria — validar que usuário não pode inserir logs com user_id diferente do seu

---

## 📖 História

**Como um** auditor de segurança,
**Eu quero** garantir que a política INSERT em `audit_logs` valida o `user_id`,
**Para que** nenhum usuário possa falsificar logs de auditoria com user_id de outro.

---

## ✅ Critérios de Aceitação

- [x] **AC1:** Política INSERT em `audit_logs` usa `WITH CHECK (user_id = auth.uid())`
- [x] **AC2:** Tentativa de inserir com user_id diferente falha com PGRST403
- [x] **AC3:** Logs legítimos (user_id = auth.uid()) podem ser inseridos com sucesso
- [ ] **AC4:** Testado em staging com múltiplos usuários

## 📝 Dev Notes

**SQL Implementation:**
```sql
-- Remove qualquer policy antiga
DROP POLICY IF EXISTS audit_logs_insert_policy ON audit_logs;

-- Criar policy com validação de user_id
CREATE POLICY audit_logs_insert_policy ON audit_logs
FOR INSERT
WITH CHECK (user_id = auth.uid());

-- Verificar
SELECT policyname, qual, with_check
FROM pg_policies
WHERE tablename = 'audit_logs' AND qual = 'INSERT';
```

**Testing Pattern:**
```typescript
// Test: User cannot insert log with different user_id
const result = await supabase
  .from('audit_logs')
  .insert({ user_id: otherUserId, action: 'test' });
expect(result.error?.code).toBe('PGRST403');

// Test: User can insert log with own user_id
const result = await supabase
  .from('audit_logs')
  .insert({ user_id: currentUserId, action: 'test' });
expect(result.error).toBeNull();
```
