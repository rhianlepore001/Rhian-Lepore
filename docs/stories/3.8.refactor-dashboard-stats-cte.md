## 📝 Dev Notes

**Problema Atual:**
- 15+ queries sequenciais cada uma chamando database separadamente
- Total execution time: 5-8 segundos
- Cada query requer round-trip ao database

**Solução (CTE):**
- Consolidar queries em 1 chamada SQL com CTEs
- Usar JOINs e window functions ao invés de loops
- Total execution time esperado: <1.5 segundos
- Uma única round-trip ao database

**Exemplo CTE Pattern:**
```sql
WITH appointment_stats AS (
  SELECT company_id, status, COUNT(*) as count
  FROM appointments
  WHERE created_at > NOW() - INTERVAL '30 days'
  GROUP BY company_id, status
),
revenue_stats AS (
  SELECT company_id, SUM(amount) as total
  FROM transactions
  WHERE created_at > NOW() - INTERVAL '30 days'
  GROUP BY company_id
)
SELECT a.*, r.total
FROM appointment_stats a
LEFT JOIN revenue_stats r ON a.company_id = r.company_id;
```

**Testing Pattern:**
```sql
-- Before (15 queries)
SELECT * FROM get_dashboard_stats_old('company-123');
-- Time: ~5000ms

-- After (1 CTE)
SELECT * FROM get_dashboard_stats_new('company-123');
-- Time: ~300ms

-- Validate
SELECT * FROM get_dashboard_stats_old('company-123')
EXCEPT
SELECT * FROM get_dashboard_stats_new('company-123');
-- Should return 0 rows (identical results)
```
